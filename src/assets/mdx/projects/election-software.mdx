Created a lightweight, asynchronous voting application using FastAPI with a MongoDB backend and deployed within a Docker container for intra-school council elections.
Supported 20+ concurrent voters with total voter count reaching 1000+ votes, and had **no discrepency** with the head count.

## Services

The project consisted of two separate services

### User Registerer

- Used for setting up Posts and Candidates
- Used ExpressJS for the backend API along with Multer for file upload handling

### Election Runner

- Handled the main election voting process.
- Consisted of password based authentication and session wise tracking for secure voting process.
- Stored vote-session objects in MongoDB which were later analysed and converted
  into a result report (see <InsiteLink to="/work/$id" slug={{ id: "elections-vote-analysis" }}>Elections Vote Analysis </InsiteLink>).
